# AWS Application Load Balancer Case Study

## 📌 Case Study Overview

An organization wants to deploy a highly available web application on AWS.
The requirement is to distribute incoming traffic across multiple EC2 instances
to ensure:

- High Availability
- Fault Tolerance
- Load Distribution

You are assigned to implement an Application Load Balancer (ALB)
to distribute traffic between two EC2 instances.

---

## 🎯 Objectives

- Launch multiple EC2 instances
- Install and configure Apache Web Server
- Configure Security Groups
- Create Target Group
- Register EC2 instances
- Create Application Load Balancer
- Validate load balancing

---

## 🧩 Tasks

1. Launch two EC2 instances using Amazon Linux
2. Configure web server using user data
3. Modify security group to allow HTTP traffic
4. Create a Target Group
5. Register EC2 instances
6. Create Application Load Balancer (Internet-facing)
7. Attach Target Group to Load Balancer
8. Validate DNS-based load distribution
